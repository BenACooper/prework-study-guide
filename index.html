HTML

  • HTML = Hypertext Markup Language. Used to create structure of webpage.
  • Used to add elements to webpages.
  • Elements are case-sensitive.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Child elements are nested within parent elements.
  • The title element is extremely important for Search Engine Optimization.
  • HTML attributes define additonal informaton about an element.
  • HTML attributes always defined in the beginning element, never the the closing element.
  • HTML Color Codes
  • Control flow is the order in which a computer executes code in a script. Achieved with "if" and "else if" statements."
  • Truthy Falsy
  • An Array is an indexed group of related items stored in a single variable. It can include multiple data types.

CSS

  • CSS stands for "Cascading Style Sheets"
  • Three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • CSS Rule: "selector {declaration}"
  • Selector defines the attribute(element or class) to which the declarations will apply.
  • A class is a semantic property assigned to elements, so they can be styled in groups.
  • Declarations contain two important components: {property: value;}
  • Multiple declarations can be included in the brackets and applied to a single selector.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS file linked in the heder element.
  • MDN Web Docs on CSS

Git

  • "Git" is a version control system that lets you manage and keep track of each version of your project.
  • "GitHub" stores our code on the cloud. It allows us to track our repo remotely and collaberate with others.
  • Introduction to GitHub
  • Git Cheat Sheet
  • Markdown Cheat Sheet
  • The default branch is called main. It is the final most updated version of the app.
  • Feature branches are copies fo the default branch. We can work on feature branches without affecting the main branch.
  • A pull request is when we merge a feature branch into the main branch. Usually peer reviewed.
  • A "commit" labels changes to files in a branch. Should have a good description to allow easy identification of individual changes.
  • IMPORTANT to frequently sync the local version of your code with the remote version in GitHub
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: commits all files in the working branch.
  • git commit -m assocaites a message with our commit
  • git checkout [branch name]: switches to the target branch
  • git pull origin [branch name]: updates our local repo with the remote repo
