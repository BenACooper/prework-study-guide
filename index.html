HTML

  • HTML = Hypertext Markup Language. Used to create structure of webpage.
  • Used to add elements to webpages.
  • Elements are case-sensitive.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Child elements are nested within parent elements.
  • The title element is extremely important for Search Engine Optimization.
  • HTML attributes define additonal informaton about an element.
  • HTML attributes always defined in the beginning element, never the the closing element.

Git

  • "Git" is a version control system that lets you manage and keep track of each version of your project.
  • "GitHub" stores our code on the cloud. It allows us to track our repo remotely and collaberate with others.
  • Introduction to GitHub
  • Git Cheat Sheet
  • Markdown Cheat Sheet
  • The default branch is called main. It is the final most updated version of the app.
  • Feature branches are copies fo the default branch. We can work on feature branches without affecting the main branch.
  • A pull request is when we merge a feature branch into the main branch. Usually peer reviewed.
  • A "commit" labels changes to files in a branch. Should have a good description to allow easy identification of individual changes.
  • IMPORTANT to frequently sync the local version of your code with the remote version in GitHub
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: commits all files in the working branch.
